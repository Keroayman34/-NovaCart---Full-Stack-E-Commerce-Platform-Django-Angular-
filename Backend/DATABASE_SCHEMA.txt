================================================================================
KEY RELATIONSHIPS
================================================================================

User Roles:
  - Admin: Full access to system, can manage all users and products
  - Seller: Can create and manage their own products
  - Customer: Can browse, purchase, and review products

Data Flow Examples:

1. Shopping Flow:
   User (Customer) → Browse Categories → View Products → Add to Cart (CartItem) 
   → Create Order (OrderItem) → Post Review

2. Inventory Flow:
   User (Seller) → Create Product → Set Price/Stock → Update via Orders

3. Notification Flow:
   Order Status Change → Create Notification → Send to User

================================================================================
CONSTRAINTS & BUSINESS RULES
================================================================================

1. Product Management:
   - Each product must have a seller (Foreign Key NOT NULL)
   - Each product must belong to a category (Foreign Key PROTECT on delete)
   - SKU must be unique
   - Price minimum: 0.01
   - Stock quantity minimum: 0

2. Cart Management:
   - Only one product per cart (enforced by unique constraint)
   - Cart can be for user OR session (not both)
   - One active cart per user

3. Order Management:
   - Each order belongs to exactly one customer
   - Order items capture price at time of purchase
   - Status transitions: pending → confirmed → shipped → delivered

4. Reviews:
   - One review per user per product
   - Rating values typically 1-5

5. Wishlists:
   - One wishlist per user (OneToOne relationship)
   - Can have multiple products
