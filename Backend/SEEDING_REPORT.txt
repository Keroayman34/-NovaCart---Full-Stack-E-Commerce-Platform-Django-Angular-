================================================================================
DATABASE SCHEMA HIGHLIGHTS
================================================================================

Key Relationships:

1. User System:
   - Each User has a role: customer, seller, or admin
   - Sellers create Products
   - Customers create Orders and Reviews

2. Product Management:
   - Products belong to Categories
   - Products are owned by Sellers
   - Products have multiple Reviews and can be in Wishlists

3. Shopping Process:
   - Customers create Carts
   - Carts contain CartItems (products)
   - Carts convert to Orders
   - Orders contain OrderItems (purchased products)

4. Engagement:
   - Customers can leave Reviews for Products
   - Customers maintain Wishlists
   - Customers receive Notifications for orders

Business Rules Enforced:

- One product per cart (unique constraint)
- One review per user per product
- One active cart per user
- One wishlist per user
- Product prices must be >= 0.01
- SKU must be unique
